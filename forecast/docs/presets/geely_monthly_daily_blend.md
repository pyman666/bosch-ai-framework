# geely_monthly_daily_blend - 月预测+日需求整合预测（模式 B）

## 适用客户
- **吉利（Geely）**
- **小鹏（Xpeng）**

---

## 业务场景

客户每月发布**月预测总量**，同时提供**每日订单明细**。系统需要：
1. 将月预测和日需求整合为每日需求
2. 考虑期初库存和在途货物（PGI/INS），计算每日净需求（日发货量）

**核心问题**：
- 日需求通常只覆盖前几周，后续需要月预测补充
- 期初库存和在途货物会抵消部分需求
- 需要计算库存余额，当余额为负时触发发货

---

## 计算逻辑

### 第一步：日需求和月预测整合

将月预测和日需求合并为每日需求：

1. **有日需求的日期**：直接取日需求值（0 值也算需求）
2. **无日需求的日期**：取月预测 / 当月天数，平摊到每天

**判断条件**：日需求是否已经覆盖到当月最后一天？
- 如果日需求已经覆盖到月底 → 后续月份使用下月预测
- 如果日需求没有覆盖到月底 → 空缺日期用月预测平摊

### 第二步：库存 Balance 计算

从期初库存开始，逐日计算库存余额：

```
Balance[第1天] = 期初库存 + 当天到货(INS/PGI) - 当天整合需求
Balance[第i天] = Balance[第i-1天] + 当天到货(INS/PGI) - 当天整合需求
```

- **期初库存**：从 `beginningInventory` 字段获取
- **INS**：预计到货量（吉利用 INS，小鹏用 PGI，系统自动识别）
- **整合需求**：第一步计算出的每日需求

### 第三步：净需求计算

```
净需求 = 如果 Balance < 0，则取 |Balance|；否则为 0
```

- 当库存余额为正时，说明库存充足，不需要发货（净需求 = 0）
- 当库存余额为负时，说明库存不足，需要发货补足（净需求 = |Balance|）
- 发货后库存余额重置为 0（因为已经补足）

### 第四步：输出日发货量

**日发货量 = 净需求**，即每日需要发货的数量。

## 输入参数

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| `monthly_forecast` | 数字 | 是 | 当月预测总量 | 8014 |
| `demand` | 日期序列 | 是 | 每日订单明细 `[{"date": "2026-01-01", "qty": 0}, ...]` | 见下方示例 |
| `beginningInventory` | 数字 | 是 | 期初库存 | 3316 |
| `pgi` | 日期序列 | 否 | PGI 在途货物明细（小鹏） | `[{"date": "2026-01-13", "qty": 576}, ...]` |

| `transportationLT` | 整数 | 否 | 运输提前量（天），默认 3 天 | 3 |

**输入数据**：

```json
    {"date": "2025-12-30", "qty": 227},
    {"date": "2025-12-31", "qty": 90},
    {"date": "2026-01-01", "qty": 0},
    {"date": "2026-01-02", "qty": 0},
    {"date": "2026-01-03", "qty": 0},
    {"date": "2026-01-04", "qty": 0},
    {"date": "2026-01-05", "qty": 178},
    {"date": "2026-01-06", "qty": 328},
    {"date": "2026-01-07", "qty": 213},
    {"date": "2026-01-08", "qty": 290},
    {"date": "2026-01-09", "qty": 420},
    {"date": "2026-01-10", "qty": 320},
    {"date": "2026-01-11", "qty": 350},
    {"date": "2026-01-12", "qty": 401},
    {"date": "2026-01-13", "qty": 402},
    {"date": "2026-01-14", "qty": 291},
    {"date": "2026-01-15", "qty": 1948}
  ],
  "beginningInventory": 3316,
  "ins": [
    {"date": "2026-01-09", "qty": 576},
    {"date": "2026-01-11", "qty": 1152},
    {"date": "2026-01-14", "qty": 576}
  ]
}
```

**计算过程**：

1. **日需求和月预测整合**：
   - 12/29 ~ 1/15：有日需求，直接取值
   - 1/16 ~ 1/31：无日需求，取月预测平摊 = 8014 / 31 ≈ 258.5/天
     - 但需扣除已有日需求的天数，实际平摊到无需求天：(8014 - 已有日需求总和) / 无需求天数

2. **库存 Balance 计算**（部分展示）：
   ```
   12/29: Balance = 3316 - 408 = 2908
   12/30: Balance = 2908 - 227 = 2681
   12/31: Balance = 2681 - 90  = 2591
   01/01: Balance = 2591 - 0   = 2591
   ...
   01/09: Balance = 1582 + 576(INS) - 420 = 1738
   ...
   01/15: Balance = 1126 - 1948 = -822  ← 库存不足！
   ```

3. **净需求**：
   - 01/15 之前：Balance 始终为正，净需求 = 0
   - 01/15：Balance = -822 < 0，净需求 = 822
   - 发货后 Balance 重置为 0

4. **最终日发货量**：
   ```
   12/29 ~ 01/14: 日发货量 = 0（库存充足）
   01/15: 日发货量 = 822（需要发货）
   后续：继续按每日整合需求扣除，不足时触发发货
   ```

## 特殊情况处理

### Case 1：只有月预测，没有日需求

如果客户只提供了月预测，没有日需求数据：
- 将月预测总量平摊到当月每一天
- 每天的需求 = 月预测 / 当月天数

### Case 2：日需求已经覆盖到下月

如果日需求数据已经覆盖到下个月：
- 当前月：按日需求取值
- 下月：如果提供了下月预测，继续平摊；否则用当前月的日均值延续

### Case 3：月预测为 0

如果月预测为 0（未提供）：
- 仅使用日需求数据
- 无日需求的日期需求为 0
- 库存 Balance 仅扣除日需求

### Case 4：INS 和 PGI 同时存在

- 优先使用 `ins` 字段（吉利客户）
- 如果 `ins` 为空，则 fallback 到 `pgi` 字段（小鹏客户）

## DSL 表达式示例

```
monthly_daily_blend(demand, 8014, beginningInventory, ins, lt=3)
```

**参数说明**：
- `demand`：日需求序列（自动从输入数据提取）
- `8014`：月预测总量
- `beginningInventory`：期初库存（自动从输入数据提取）
- `ins`：预计到货序列（自动从输入数据提取，fallback 到 pgi）
- `lt=3`：运输提前量（可选，默认 3 天）

**组合使用**：
```
// 先计算库存余额，再计算净需求
net_demand(balance(beginningInventory, demand, pgi))
```

## Python 脚本示例

```python
def forecast(record):
    monthly = float(record.get("monthly_forecast", 0))
    daily = [float(x.get("qty", 0)) for x in record.get("demand", [])]
    begin_inv = float(record.get("beginningInventory", 0))
    ins = [float(x.get("qty", 0)) for x in record.get("ins", record.get("pgi", []))]
    
    n = len(daily)
    if n == 0:
        return []
    
    # 月预测平摊
    daily_avg = monthly / max(n, 1) if monthly > 0 else 0
    
    # 整合需求（有日需求取日需求，否则取平摊值）
    blended = [daily[i] if i < len(daily) else daily_avg for i in range(n)]
    
    # 库存 Balance
    bal = begin_inv
    result = []
    for i in range(n):
        inflow = ins[i] if i < len(ins) else 0
        bal = bal + inflow - blended[i]
        net = max(0, -bal) if bal < 0 else 0
        result.append({"date": record["demand"][i]["date"], "qty": round(net)})
        bal = max(0, bal)
    
    return result
```

## 常见问题

**Q1：日需求为 0 算不算"有需求"？**  
A：算。只要该日期在 `demand` 数组中存在（有 date），即使 qty=0 也算"有需求"。只有完全缺失该日期才算"无需求"。

**Q2：月预测平摊到每天，会不会不准确？**  
A：这是行业通用做法。日需求通常只覆盖前 1-2 周，后续用月预测平摊是合理的近似。如果需要更精确，建议提供更详细的日需求数据。

**Q3：Balance 变为负后为什么要重置为 0？**  
A：因为净需求 = |Balance| 意味着已经安排了等量发货。发货后库存应该恢复到 0（刚好满足需求），不会继续为负。

**Q4：INS 和 PGI 有什么区别？**  
A：INS = 预计到货（In-transit），PGI = Post Goods Issue（已发货在途）。两者本质上都是在途货物，只是不同客户的叫法不同。系统自动识别：优先用 `ins`，没有则用 `pgi`。

**Q5：跨月怎么处理？**  
A：使用 dict 格式的 `monthly_forecast` 支持多月预测（对齐 Excel Logic sample）：
- 日期轴从最早需求日（或首月 1 号，取早者）到末月最后一天
- 每月的 fill_rate 独立计算：`(月预测 − 当月 demand 合计) / 当月剩余天数`
- 首月之前的日期（预预测期）：有 demand 取 demand，无则 0
- 示例：`{"2026-01": 8014, "2026-02": 5471}` 生成 12/29~02/28 共 62 天
- float 格式向后兼容：视为单月 forecast 应用于 first_day 所在日历月
